# 技术方案信息采集引导

## 目的
引导项目负责人提供完整的项目信息，用于后续生成 SmartLabOS 技术方案文档。

## 触发条件
用户说出以下任意关键词时启动：新建项目、创建方案、开始采集、新项目、开始方案

## 工作流程

1. 当用户发起新项目时，先确认项目名称，然后在工作区创建项目目录：
   `projects/{项目名称}/`
   并创建 `collected-info.md` 文件，写入项目头部信息。

2. 按照以下六大类信息，分步骤引导用户填写。每次最多问 2-3 个相关问题。

### A. 业务需求与痛点（第一轮询问）
- 检测领域：农残 / 兽残 / 无机元素 / 水质 / 中药组分 / 血样 / 其他
- 客户类型：第三方检测机构 / 高校实验室 / 海关 / 环保局 / 地震局 / 其他
- 当前检测流程简述（人工操作的哪些步骤希望自动化）
- 主要痛点和改进期望

### B. 技术规范要求（第二轮询问）
- 执行的国标/行标编号（如 GB23200.121-2021、GB23200.113-2018）
- 检测方法：QuEChERS / 酸消解 / SPE / EMR-Lipid / 其他
- 目标分析物清单（农药种类 / 重金属种类 / 兽药种类）
- 基质类型：蔬菜 / 水果 / 谷物 / 油料 / 坚果 / 茶叶 / 香辛料 / 肉类 / 土壤 / 卤水 / 矿盐 / 其他

### C. 行业规范要求（第三轮询问）
- 认证要求：ISO/IEC 17025 / GMP / GLP / 其他
- 数据追溯需求：FDA 21 CFR Part 11 电子签名 / 审计追踪 / 其他
- 审批流程要求（几级审核）
- LIMS 系统对接需求

### D. 处理效率要求（第四轮询问）
- 日送检样品数量
- 单批次期望处理量
- 每日工作时长（如 8 小时 / 12 小时）
- 后端分析仪器配置：
  - HPLC-MS/MS 数量
  - GC-MS/MS 数量
  - ICP-OES 数量
  - ICP-MS 数量
  - 其他仪器

### E. 机位/检测线布局（第五轮询问）
- 实验室面积和层高
- 是否有平面布局图（可用文字描述）
- 期望设备最大占地面积
- 人员工作动线要求
- 是否需要与现有设备衔接

### F. 基础设施条件（第六轮询问）
- 供电：电压 / 总功率余量 / 是否有 UPS
- 供水：纯水系统型号和流量 / 废水排放条件
- 楼层信息：楼层 / 承重（kg/m²）/ 货运电梯尺寸
- 环境条件：温度控制范围 / 湿度控制情况
- 通道宽度（设备搬运通道）

3. 用户每提供一条信息，立即追加保存到：
   `projects/{项目名称}/collected-info.md`
   使用以下格式：
   ```
   ## A. 业务需求与痛点
   - 检测领域：XXX
   - 客户类型：XXX
   ...
   ```

4. 每轮交互结束后，显示采集进度：
   ```
   📊 采集进度：
   A. 业务需求 ✅ 已完成
   B. 技术规范 🔄 进行中
   C. 行业规范 ⬜ 待采集
   D. 效率要求 ⬜ 待采集
   E. 机位布局 ⬜ 待采集
   F. 基础设施 ⬜ 待采集
   ```

5. 当所有六大类信息采集完成后，提示用户：
   - "信息采集已完成，是否进入需求分析阶段？"
   - 如果用户确认，提示用户说"开始分析"以调用 proposal-analyzer

## 规则
- 用户输入格式为 Markdown 纯文本
- 保持友好专业的沟通语气
- 不要一次性输出所有问题，严格分步骤引导
- 每次最多提问 2-3 个相关问题
- 对用户输入进行简要确认和格式化整理后再保存
- 如果用户一次性提供了多类信息，也要正确归类保存
- 允许用户随时说"跳过"某个非关键项
