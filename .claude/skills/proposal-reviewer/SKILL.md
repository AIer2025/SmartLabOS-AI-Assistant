# 技术方案审查

## 目的
对生成的技术方案文档进行全面质量检查。

## 触发条件
用户说出：审查方案、检查方案、方案审查

## 前置条件
项目目录下必须存在 `technical-proposal.md`。

## 检查维度

### 1. 完整性检查
逐一检查以下章节是否存在且有实质内容：
- [ ] 系统简介
- [ ] 工艺流程
- [ ] 系统组成（含各功能单元参数）
- [ ] 软件系统
- [ ] 系统效率分析
- [ ] 现场要求
- [ ] 质量保证与服务

### 2. 参数一致性检查
对照知识库文件（`references/product-specs.md` 和 `unit-catalog.md`），检查：
- 方案中的功能单元参数是否与产品规格一致
- 精度、转速、温度等数值是否在有效范围内
- 没有出现自相矛盾的参数

### 3. 国标合规性检查
对照 `references/standards-library.md`，检查：
- 引用的国标编号是否正确
- 工艺步骤是否符合国标要求
- 关键参数（称样量、溶剂体积、离心条件）是否与国标一致

### 4. 通量合理性检查
- 效率计算的数字是否自洽
- 日处理量是否满足客户需求
- 后端仪器供给是否合理

### 5. 客户需求匹配检查
对照 `collected-info.md`，检查：
- 方案是否覆盖了客户所有检测领域
- 基质类型是否全部包含
- 效率要求是否满足
- 基础设施条件是否匹配

## 输出格式

生成审查报告保存到 `projects/{项目名称}/review-report.md`，格式：

```markdown
# 技术方案审查报告
项目：{项目名称}
审查时间：{当前日期}

## 审查结果汇总
- 通过项：X 项
- 警告项：Y 项
- 需修改项：Z 项

## 详细审查结果

### 1. 完整性检查
✅ 系统简介 — 内容完整
✅ 工艺流程 — 内容完整
⚠️ 系统组成 — 缺少过滤单元参数
❌ 现场要求 — 供水需求未填写
...

### 2. 参数一致性检查
...

### 3. 国标合规性检查
...

### 4. 通量合理性检查
...

### 5. 客户需求匹配检查
...

## 修改建议
1. {具体修改建议}
2. {具体修改建议}
...
```

## 规则
- 审查必须逐项进行，不可跳过
- 每项给出明确的 ✅ ⚠️ ❌ 判定
- 发现问题时必须给出具体修改建议
- 审查完成后，告知用户结果摘要，并建议：
  - 如果有 ❌ 项：建议修改后重新生成
  - 如果只有 ⚠️ 项：建议人工确认后可用
  - 如果全部 ✅：方案可交付
