# 技术方案文档生成

## 目的
生成符合百泉聚兴标准模板的完整技术方案文档（Markdown 格式）。

## 触发条件
用户说出：生成方案、出方案、写方案

## 前置条件
项目目录下必须存在 `analysis-report.md`。
如果不存在，提示用户先完成需求分析。

## 文档结构

按照以下章节结构生成完整技术方案。参考 `references/proposal-template.md` 获取格式规范。

```markdown

百泉聚兴（北京）科技有限公司

---
```

## 工作流程

1. 读取 `projects/{项目名称}/analysis-report.md`
2. 读取 `projects/{项目名称}/collected-info.md`
3. 读取以下知识库文件：
   - `references/proposal-template.md`
   - `references/product-specs.md`
   - `references/unit-catalog.md`
   - `references/process-quechers.md` 或 `process-inorganic.md`
4. 按章节顺序逐步生成内容
5. 完成后保存到：`projects/{项目名称}/technical-proposal.md`
6. 通知用户方案已生成，建议说"审查方案"进行质量检查

## 规则
- 严格按照模板结构组织文档，不遗漏章节
- 技术参数必须引用自知识库，不可编造
- 工艺流程必须符合对应国标要求
- 用专业但清晰的中文书写
- 表格、参数列表格式规范统一
- 每个功能单元的参数要完整列出
- 系统简介要根据客户的具体需求定制，不要照搬通用描述
