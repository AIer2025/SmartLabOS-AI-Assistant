# 技术方案需求分析

## 目的
基于采集到的客户信息，匹配最合适的工艺流程和设备配置方案。

## 触发条件
用户说出：开始分析、分析需求、需求分析

## 前置条件
项目目录下必须存在 `collected-info.md`，且至少 A（业务需求）和 B（技术规范）已填写。
如果信息不足，提示用户先完成信息采集。

## 工作流程

1. 读取 `projects/{项目名称}/collected-info.md`

2. 读取知识库文档（位于 references/ 目录）：
   - `product-specs.md` — SmartLabOS 产品功能单元参数
   - `process-quechers.md` — QuEChERS 农残/兽残工艺流程
   - `process-inorganic.md` — 无机元素前处理流程
   - `standards-library.md` — 国标法规关键要求
   - `unit-catalog.md` — 硬件单元型号与适用场景

3. 执行以下六项分析：

   **a. 检测领域识别**
   根据客户的检测领域和基质类型，确定主工艺路线：
   - 农残 → QuEChERS (GB23200.113-2018 / GB23200.121-2021)
   - 兽残 → Agilent EMR-Lipid / 高通量兽残筛查
   - 无机元素 → 酸消解 + ICP
   - 其他领域 → 标记为需要定制方案

   **b. 国标匹配**
   根据客户指定的国标编号和基质类型，确认具体的前处理方法和关键参数：
   - 试样量、溶剂种类和体积
   - 盐包组成和用量
   - 离心转速和时间
   - SPE 柱规格和流速
   - 浓缩温度和时间

   **c. 通量计算**
   根据日检量、工作时长、单批次处理量计算：
   - 每日需要的批次数 = 日检量 ÷ 单批次处理量
   - 全流程单批次耗时估算
   - 是否能满足后端仪器的供样需求
   - 如果通量不足，建议增加通道数或调整批次策略

   **d. 设备选型**
   列出需要的功能单元清单及推荐参数设置：
   - 加液涡旋单元（通道数、加液种类）
   - 加盐涡旋单元（盐包种类）
   - 振荡单元 / 离心单元 / 移液单元
   - SPE 单元 / 浓缩单元 / 加内标单元 / 过滤单元 / 超声单元
   - 每个单元的关键参数设置值

   **e. 基础设施校验**
   核对客户现场条件是否满足系统需求：
   - 供电：系统总功率（约 6-8kVA）vs 客户可用功率
   - 设备尺寸 vs 客户场地面积
   - 楼层承重 vs 设备重量
   - 供水 / 排水条件

   **f. 风险与缺项识别**
   标记以下情况：
   - ❌ 信息严重不足，无法继续的项
   - ⚠️ 条件不完全匹配，需要确认的项
   - 💡 建议优化的项

4. 将分析结果保存到：
   `projects/{项目名称}/analysis-report.md`

5. 向用户展示分析摘要，包含：
   - 推荐的工艺路线
   - 关键设备清单
   - 通量是否满足
   - 风险项列表
   等待用户确认或调整后，提示说"生成方案"以进入下一阶段。

## 规则
- 所有技术参数必须来自知识库文档，不可编造
- 通量计算必须给出详细推导过程和数字
- 信息不足时明确列出需要补充的具体项
- 风险项使用 ❌ ⚠️ 💡 图标醒目标注
- 分析报告使用结构化 Markdown 格式
