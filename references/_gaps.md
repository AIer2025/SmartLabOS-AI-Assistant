## 一、逐文件缺口盘点

### 1. [process-inorganic.md](process-inorganic.md)

**现状**：系统概述、整体参数（220V/8kVA/W3600×D1200×H2150）、3 类样品（卤水/矿盐/土壤）、8 个功能单元参数、线边仓库配置齐全。
**头部 ⚠️ 声明**：需根据「无机元素测试全流程自动化样本前处理技术方案.pdf」补充完善。
**缺口**：
- **无分步骤工艺流程** — 对比 process-quechers.md 第 3/4 节的粒度，本文件没有"称样→加酸→消解→定容→稀释"这类具体可执行步骤
- **未引用任何国标** — 砷/镉/铅/汞常用的 GB 5009.11/12/15/17、HJ 803（土壤）、GB/T 5009.268 均未提及
- **基质差异化参数空白** — 卤水 vs 矿盐 vs 土壤在称样量、消解酸体系、消解曲线上的差异未列出
- **石墨消解程序升温曲线缺失** — 仅给出温度范围 30-200°C，无阶段温度/保持时间/赶酸参数
- **与后端 ICP-OES/MS 的接样衔接未描述** — 定容体积、稀释倍数、进样管规格未说明

### 2. [process-quechers.md](process-quechers.md)

**现状**：GB23200.113-2018、GB23200.121-2021 植物源流程完整；Agilent 多兽药残留方法完整；10 个功能单元参数齐全。
**头部 ⚠️ 声明**：需根据「食品安全农兽残检测样品前处理系统技术方案.pdf」补充完善。
**缺口**：
- **仅覆盖植物源** — 缺动物源性食品流程（GB 31658.17 肉类、GB 31660.5 蛋奶/水产）
- **缺茶叶、香辛料基质流程** — standards-library 提及 GB23200.121 覆盖这两类，但正文未写
- **缺海关/进出口专用方法** — SN/T 系列（如 SN/T 0127、SN/T 1982）完全未提
- **缺液体基质 QuEChERS 变体** — 葡萄酒/果汁/蜂蜜无专用流程
- **缺高脂/高色素/干制样品的额外净化** — C18、GCB、EMR-Lipid 的选用决策树未给出

### 3. [product-specs.md](product-specs.md)

**现状**：微服务架构（5 层 + 11 服务）、技术栈（.NET 8 / SqlSugar / MySQL-达梦 / 麒麟）、核心模块（LIMS/MES/WMS/EAM）、性能指标、集成接口均有框架。
**头部 ⚠️ 声明**：需将「百泉聚兴智慧实验室 SmartLabOS 系统技术规格书.pdf」关键内容填入。
**缺口**：
- **缺部署架构图** — 单机 / 分布式 / 信创三种模式的硬件推荐配置
- **缺等保 2.0 三级合规细节** — 与 standards-library 的 GB/T 22239 双向缺失
- **缺 LIMS/MES 详细数据流图** — 仅列出模块功能，未描绘业务流转
- **API 清单停在表格** — 未给出字段/协议/示例报文
- **缺第三方仪器兼容矩阵** — 40+ Modbus 驱动的品牌/型号/协议清单未展开
- **缺灾备与数据归档方案** — "在线 ≥3 年、归档 ≥10 年"如何实现未说明

### 4. [proposal-template.md](proposal-template.md)

## 二、海关农残检测系统提案——预计卡点

海关场景典型特征：进出口食品农残筛查、**植物源 + 动物源双基质**、高通量、报告需符合 CIQ 格式、须引用 **GB 2763 限量值** 和 **SN/T 系列** 行标。

按五阶段工作流推演：

| 阶段 | 环节 | 是否卡住 | 原因 |
|------|------|---------|------|
| 1. 创建提案项目 | 建目录/初始化 | 否 | 纯结构操作，proposal-workflow Skill 可独立完成 |
| 2. 提案信息收集 | 采样/通量/基质问卷 | 部分 | proposal-collector 能问到基础需求，但在"适用标准选型"子问题上无法给出 SN/T 备选 |
| **3. 提案初稿生成** | **工艺流程章节** | **是（硬卡）** | process-quechers.md 无动物源流程；进口肉类/乳制品/水产无法落笔 |
| 3. 提案初稿生成 | 标准引用章节 | **是（硬卡）** | standards-library.md 缺 GB 2763、SN/T 0127、SN/T 1982、GB 31658，无法合规引用 |
| 3. 提案初稿生成 | 系统组成章节 | 部分 | unit-catalog.md 缺均质/研磨前端单元，动物组织前处理链条写不全 |
| 3. 提案初稿生成 | 系统效率分析 | 否 | 现有功能单元参数可推算 |
| 3. 提案初稿生成 | 软件系统章节 | 部分 | product-specs.md 缺 CIQ 报告模板、海关单证接口描述 |
| **4. 审核及修改** | 参数一致性/国标合规 | **是（审核反阻塞）** | proposal-reviewer 会发现方案引用的 SN/T 标准未在 standards-library 登记，依 SOUL.md"不确定时诚实说明"原则触发停止 |
| 5. 正式发布 | 定稿输出 | 取决于前序 | 若 3、4 阶段已补齐则不卡 |

**结论**：最早在阶段 3 的"工艺流程"和"标准引用"硬卡；阶段 4 审核会第二次触发阻塞。即便跳过阶段 4，阶段 3 也过不了。
