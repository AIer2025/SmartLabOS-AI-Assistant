## 1. 必要前置工作（首次启用必做）

### 1.1 Python 环境（已确认可用）

**生效路径**：`D:\ProgramData\anaconda3\envs\py39\python.exe`（Python 3.9.18 + pyyaml 6.0.1）

启用方式两选一：

**方式 A — 每次手动激活（推荐用于交互调试）**

```powershell
# 在项目根目录
conda activate py39
python tools/validate_solution.py references/solutions/SOL-QUECHERS-MID.md
```

**方式 B — 永久把 py39 加入用户 PATH**（Pre-commit 钩子需要这个）

git pre-commit 钩子是 `bash` 脚本，会调用 `python` / `python3` 命令。如果终端激活的不是 py39，
钩子会找不到 Python（或找到 Microsoft Store 别名）。两个解决方案：

- 在 Git Bash 启动文件（`~/.bashrc` 或 `~/.bash_profile`）中加入：
  ```bash
  export PATH="/d/ProgramData/anaconda3/envs/py39:$PATH"
  ```
- 或全局修改用户 PATH 环境变量，把 `D:\ProgramData\anaconda3\envs\py39` 与
  `D:\ProgramData\anaconda3\envs\py39\Scripts` 加在 Microsoft Store 别名之前。

> ⚠️ **千万避开 Microsoft Store 的 python/python3 别名**：在 Windows 终端直接敲 `python` 会跳到 Store 安装页（exit 49）。检测命令 `where python` 如果第一行是 `WindowsApps` 路径，说明别名优先，需要调整 PATH 顺序。

### 1.2 安装 pre-commit 钩子（每次新克隆都要做）

**当前状态**：钩子源已在 [git-hooks/pre-commit](../git-hooks/pre-commit)，但**尚未安装到 `.git/hooks/`**。

```bash

```

安装后 `.git/hooks/pre-commit` 出现，每次 `git commit` 自动校验改动到的卡片。

### 1.3 验证 GitHub Actions 首次执行

推送 commit `2021455` 后 CI 应该已经跑过。访问：
[github.com/AIer2025/SmartLabOS-AI-Assistant/actions](https://github.com/AIer2025/SmartLabOS-AI-Assistant/actions)

1. `validate-solutions` — 全库方案校验

如果 `check-indexes` 红色，**第一可能原因**是 build_indexes 输出与 commit 中的 `_index.md` 不

## 7. 注意事项汇编

### 7.1 Python 调用相关

- ⚠️ Windows 默认 `python` 是 Microsoft Store 别名，运行时跳转到 Store 安装页（exit 49）。**所有 tools/ 脚本调用都需要先 `conda activate py39` 或用全路径**
- ⚠️ Windows 控制台默认 GBK 编码，无法输出 ✓ ✗ 等 Unicode 字符。需要 `set PYTHONIOENCODING=utf-8` 或 `chcp 65001`。check-all.bat 与 pre-commit 钩子已经预置，**手动跑脚本时也要预置**
- ⚠️ `build_indexes.py` 不带参数时从 cwd 自动定位 kb_root，但项目根没有 platforms/ modules/ 同级目录，会失败。**手动跑时务必加 `--kb-root references`**
- `validate_solution.py` 与 `recompute_resource_summary.py` 自动从 SOL 文件向上找 kb_root，**不需要 --kb-root**

### 7.2 卡片编辑相关

- 改了模块的关键字段（功率、`compatible_platforms` 等）后，**必须**跑全库方案校验，否则
  下游方案可能悄悄被打破。pre-commit 钩子会自动做这件事
- 方案卡片的 `resource_summary` **绝对不要手填**，永远用 recompute 脚本算，否则容易出错
- `_index.md` 由脚本自动生成，**不要手工编辑**，会被下次 build_indexes 覆盖
- 文件名以下划线开头（`_index.md`、`_categories.md`）的文件被视作"基础设施"，
  validate_solution 和 build_indexes 会跳过它们，不会当成卡片再次处理

### 7.3 Git / GitHub 相关

- 远程仓库 SSH：`git@github.com:AIer2025/SmartLabOS-AI-Assistant.git`
- 主分支：`main`
- 提交前 pre-commit 钩子会校验卡片，FAIL 拒绝提交。应急绕过（不推荐）：`git commit --no-verify`
- CI 工作流仅在改动 `references/**` 或 `tools/**` 时触发，节省 Actions 配额
- 推送大批量文件时会出现 LF/CRLF 警告，无害，加 `.gitattributes` 后消失（见 §6.2）

### 7.4 业务/合规相关（节选自 CLAUDE.md）

- 所有技术参数必须来自 references/ 知识库，**严禁编造**
- 工艺流程必须严格遵循对应国标
- 涉及报价问题统一回复"请联系销售团队获取报价"
- 不确定时诚实说明并列出待补充信息，不要硬编

### 7.5 文档同步

- README 的"已落地的卡片库"表里，卡片数是当前 seed 数（1/6/1）。**真实数据填进去后要更新这个表的卡片数**
